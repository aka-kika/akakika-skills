# Examples — project-catch-up

## Before you code — samples

**Safe return**
> **Headline:** Nothing urgent—you're on a supported Next.js 15 line; one optional Cursor skill could speed up forms.
>
> **Do these first:**
> 1. Run `npm install` in case lockfile changed on another machine.
> 2. Skim the Must know row about environment variables (none blocking).

**Pause for upgrade**
> **Headline:** Stop before big UI work—Expo SDK 52 migration guide fixes breaking changes you'll hit immediately.
>
> **Do these first:**
> 1. Read [Expo SDK 52 changelog](https://expo.dev/changelog/sdk-52) (15 min).
> 2. Run the official upgrade command from that page.
> 3. Then continue your habit-screen feature.

**Opportunity**
> **Headline:** You were about to hand-roll PDF export—Nutrient or a Vercel template now covers 80% of it.
>
> **Do these first:**
> 1. Try the tool in a throwaway branch before deleting your export code.

## Cursor skills & MCP row sample

| Name | What it helps with | How to try |
|------|-------------------|------------|
| `supabase` (Cursor plugin skill) | Auth + DB patterns for your stack | Enable Supabase plugin in Cursor |
| Playwright MCP | Browser testing without writing boilerplate | Add server in Cursor MCP settings |

**Learn:** An *MCP server* is a small bridge that lets Cursor talk to a tool (browser, database, Notion)—like a plugin with structured actions.
